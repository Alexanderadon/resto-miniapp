# Спецификация экранов — Telegram Mini App «заказ из ресторана»

Вьюпорт: телефон ~390px внутри Telegram WebView. Язык — русский. Валюта — ₸ (формат `2 400 ₸`, неразрывный тонкий пробел между тысячами и перед ₸).

## Общие паттерны

- **Touch-таргеты ≥44×44px** — все кнопки, чипсы, степперы, строки списка (утилита `tap-target`).
- **Sticky-элементы снизу** всегда с `pb-safe-3` (safe-area iPhone).
- **Offline:** глобальный слушатель `online/offline` → баннер `OfflineBanner` под шапкой («Нет соединения — показываем сохранённое меню»), кнопки действий с сетью — disabled. Каталог из кэша остаётся доступен.
- **Error сети (запрос упал):** компонент `ErrorState` — иллюстрация, «Не удалось загрузить», кнопка «Повторить» (44px). При ошибке — `notificationOccurred('error')`.
- **Haptics:** только на осмысленных действиях, не на каждом скролле. Используется `Telegram.WebApp.HapticFeedback`.
- **Тема:** при `Telegram.WebApp.colorScheme === 'dark'` ставим `data-theme="dark"` на `<html>`; подписка на `themeChanged`.

## 1. Каталог (`/`)

**Layout** (сверху вниз):
1. Шапка (sticky, фон `header`): название ресторана + подзаголовок «Самовывоз ・ Алматы».
2. Горизонтальная лента категорий-чипсов (sticky под шапкой, `overflow-x-auto scrollbar-none`, `px-4 gap-2`): «Все», «Салаты», «Горячее», «Супы», «Десерты», «Напитки». Активный чипс — `bg-brand text-on-brand`, неактивный — `bg-surface text-ink border border-line`. Высота 40px + вертикальный паддинг ленты = таргет ≥44px.
3. Сетка блюд: 2 колонки, `grid-cols-2 gap-3 px-4`, скролл при выборе категории — плавный к началу секции.
4. **Sticky cart-bar** снизу (появляется при ≥1 позиции, слайд снизу 200ms): `bg-brand text-on-brand rounded-button mx-4 shadow-bar pb-safe`-обёртка; слева «Корзина ・ 3», справа сумма «5 200 ₸». Высота 52px. Тап → Корзина.

**Карточка блюда** (`DishCard`, radius `card`, `bg-surface shadow-card`):
- Фото 1:1 сверху (lazy, `object-cover`, скруглены только верхние углы), плейсхолдер-заглушка с иконкой блюда до загрузки.
- Название (2 строки max, ellipsis), вес/объём (`text-caption text-muted`).
- Нижняя строка: цена (`text-price`) + кнопка **«+»** (`IconButton` 44×44, `bg-brand-soft text-brand`, `aria-label="Добавить {название}"`). Если позиция уже в корзине — вместо «+» мини-`Stepper` (− count +).
- Тап по карточке (не по «+») → bottom sheet блюда.

**Состояния:**
- **Loading:** skeleton — лента из 5 чипсов-скелетонов + сетка 6 карточек (прямоугольник фото + 2 строки текста + строка цены), утилита `skeleton`.
- **Empty** (категория пуста): `EmptyState` — «В этой категории пока пусто», кнопка «Смотреть всё меню».
- **Error сети:** `ErrorState` на месте сетки, чипсы остаются.
- **Offline:** баннер + каталог из кэша; кнопки «+» работают (корзина локальная).

**Haptics:**
- «+» / изменение степпера → `impactOccurred('light')`.
- Переключение категории → `selectionChanged()`.
- Появление cart-bar (первое добавление) → `impactOccurred('medium')`.

## 2. Карточка блюда (bottom sheet)

**Layout** (`BottomSheet`, `rounded-t-[--radius-sheet] bg-surface shadow-sheet`, затемнение бэкдропа 40%, закрытие: свайп вниз / тап по бэкдропу / крестик):
1. Грабер-полоска сверху по центру.
2. Фото 4:3 на всю ширину шита.
3. Контент `px-4`: название (`text-title`), вес и ккал (`text-caption text-muted`), описание/состав (2–4 строки), бейджи («Острое», «Вегетарианское» — `Badge`).
4. Sticky-низ шита (`pb-safe-3`, фон `surface`, тонкая линия сверху): слева `Stepper` (44px кнопки − / +, count `tabular-nums`), справа кнопка-`Button` primary на всю оставшуюся ширину: **«В корзину ・ 2 400 ₸»** — сумма пересчитывается от степпера. Если блюдо уже в корзине — «Обновить ・ 3 600 ₸».

**Состояния:**
- **Loading:** открывается мгновенно с данными из каталога (название/цена уже есть), догружаются только детали: skeleton на описание и фото-блюр из превью.
- **Error сети:** детали не догрузились — показываем базовые данные + строка «Описание недоступно» (не блокируем добавление).
- **Offline:** то же — добавление в корзину работает (локальный стейт).
- Блюдо закончилось (`out of stock`): фото ч/б, кнопка disabled «Закончилось», степпер скрыт.

**Haptics:**
- Открытие шита → `impactOccurred('light')`.
- − / + степпера → `impactOccurred('light')`; на минимуме/максимуме (например, 20 шт.) → `notificationOccurred('warning')` один раз.
- «В корзину» → `notificationOccurred('success')`, шит закрывается, cart-bar подпрыгивает (scale 1.03 → 1).

## 3. Корзина (`/cart`)

**Layout:**
1. Шапка: «Корзина» (`text-title`) + текстовая кнопка «Очистить» справа (danger, с confirm-шитом «Удалить все позиции?»).
2. Список позиций (`CartItemRow`, фон `surface`, radius `card`, `gap-2`): миниатюра 56×56, название + цена за шт. (`text-caption text-muted`), справа `Stepper` 44px и сумма строки. Удаление: **свайп влево** открывает красную зону с корзинкой (`SwipeableRow`) **и** дублирующий крестик у строки — для тех, кто не свайпает. Уменьшение степпера до 0 = удаление с confirm-подпрыгиванием строки.
3. Блок итога (`Card`): «Сумма — 5 200 ₸», строка «Самовывоз — бесплатно», разделитель, **«Итого — 5 200 ₸»** (`text-price` полужирный).
4. Sticky-кнопка снизу: `Button` primary во всю ширину `mx-4 pb-safe-3` — **«Оформить ・ 5 200 ₸»**.

**Состояния:**
- **Loading:** корзина локальная — скелетоны только на миниатюры фото; при ревалидации цен с сервера — тонкий прогресс под шапкой.
- **Empty:** `EmptyState` — иллюстрация пустой тарелки, «В корзине пусто», подпись «Загляните в меню — там вкусно», кнопка «В меню». Sticky-кнопки нет.
- **Error сети** (ревалидация цен упала): корзина показывается, тост «Не удалось проверить цены — суммы могут измениться», кнопка «Оформить» активна.
- **Offline:** баннер, «Оформить» disabled с подписью «Нужно соединение для оформления».
- Позиция стала недоступна: строка приглушена, бейдж «Закончилось», строка «Удалить недоступные» над итогом.

**Haptics:**
- Степпер → `impactOccurred('light')`.
- Свайп достиг порога удаления → `impactOccurred('medium')`; удаление → `notificationOccurred('warning')`.
- «Очистить» подтверждено → `notificationOccured('warning')`.

## 4. Оформление (`/checkout`)

**Layout** — одна колонка, секции-карточки `Card` с заголовками (`text-caption text-muted uppercase`):
1. **Контакты:** `Input` «Имя» — предзаполнено из `Telegram.WebApp.initDataUnsafe.user.first_name`; `PhoneInput` «Телефон» — маска `+7 (___) ___-__-__`, `inputmode="tel"`, префикс +7 фиксирован.
2. **Время самовывоза** (`TimeSlotPicker`): горизонтальные чипсы-слоты, первый — «Через 30 мин (14:35)», дальше шаг 15 мин до конца рабочего дня: «14:45», «15:00»… Активный слот — `bg-brand text-on-brand`. Слоты вне графика ресторана не генерируются; если до закрытия <30 мин — секция заменяется на notice «Сегодня уже не успеем — откроемся завтра в 10:00».
3. **Оплата** (`RadioGroup`, строки ≥44px, radio справа): «Наличными при получении» / «Картой онлайн». При выборе карты — подпись «Оплата после подтверждения заказа».
4. **Комментарий:** `Textarea` «Например: без лука», autogrow до 4 строк, счётчик 200 симв.
5. Итог: строка «Итого ・ 5 200 ₸».
6. Sticky-кнопка `pb-safe-3`: **«Заказать ・ 5 200 ₸»** (при оплате картой — «Перейти к оплате ・ 5 200 ₸»).

**Валидация:** инлайн, по blur и по сабмиту: имя ≥2 символов, телефон — полная маска, слот выбран (первый — по умолчанию). Ошибка — рамка `danger` + текст под полем, скролл к первому невалидному полю.

**Состояния:**
- **Loading:** сабмит → кнопка в состоянии loading (спиннер + «Отправляем…»), поля disabled. Слоты грузятся — 4 чипса-скелетона.
- **Empty:** прямой заход с пустой корзиной → redirect в Корзину (empty-state).
- **Error сети** (сабмит упал): кнопка возвращается, тост-ошибка «Не удалось отправить заказ. Попробуйте ещё раз» + `notificationOccurred('error')`. Данные формы сохраняются (draft в localStorage).
- **Offline:** кнопка disabled + баннер; форма редактируется, черновик сохраняется.

**Haptics:**
- Выбор слота / способа оплаты → `selectionChanged()`.
- Ошибка валидации при сабмите → `notificationOccurred('error')`.
- Успешный сабмит → `notificationOccurred('success')` (уже на экране Успех).

## 5. Успех (`/order/[id]`)

**Layout** (по центру, выдержанный, без конфетти-перегруза):
1. Иконка-галочка в круге `bg-success-soft text-success` (анимация scale-in 300ms).
2. «Заказ принят!» (`text-title`), под ним — **«№ 1042»** крупно.
3. `Card` — чек: список позиций (название × кол-во — сумма), разделитель, «Итого — 5 200 ₸», строки «Самовывоз ・ 15:00», «Оплата ・ наличными», адрес ресторана.
4. Статус (`StatusBadge` + текст): «Готовим ваш заказ» — обновляется по polling'у (Новый → Готовится → Готов к выдаче).
5. Кнопки: primary «Вернуться в меню», secondary текстовая «Показать номер бармену» (номер крупно на весь экран — оффлайн-friendly).

**Состояния:**
- **Loading:** переход с оформления — данные уже в памяти, скелетон только если открыт по прямой ссылке (skeleton чека).
- **Error сети / Offline:** чек показывается из локальной копии, статус — «Статус обновится при появлении сети», без кнопки Повторить-спама (ретрай авто).
- Заказ не найден (битый id): `ErrorState` «Заказ не найден» + «В меню».

**Haptics:**
- Вход на экран → `notificationOccurred('success')` (один раз).
- Смена статуса на «Готов к выдаче» при открытом экране → `notificationOccurred('success')`.

## 6. Админ (`/admin`) — desktop-friendly

Отдельный лэйаут: max-width 1200px, центр, работает и на телефоне (таблица схлопывается в карточки при <768px). Доступ по Telegram-ID из allowlist; чужим — `ErrorState` «Нет доступа».

**Layout:**
1. Шапка: «Заказы» + счётчик активных + переключатель авто-обновления (polling 15с).
2. Фильтр по статусу — сегменты/чипсы: «Все», «Новые» (badge с числом), «Готовятся», «Готовы», «Выданы», «Отменены».
3. **Таблица заказов** (desktop): колонки № / Время слота / Имя + телефон (телефон — `tel:` ссылка) / Состав (свёрнуто, «3 позиции») / Сумма / Оплата / Статус (`StatusBadge`) / Действия. Строка кликабельна → раскрывающаяся карточка заказа.
4. **Карточка заказа** (раскрытие строки или отдельная панель): полный состав с количеством, комментарий клиента (выделен `brand-soft`), время создания, кнопки смены статуса — только валидные переходы: Новый → [Принять] → Готовится → [Готов] → Готов → [Выдан]; отдельно [Отменить] (danger, с confirm). Кнопки ≥44px.
5. Новый заказ при polling'е — строка подсвечивается `brand-soft` 2с + звуковой пинг (отключаемый).

**Состояния:**
- **Loading:** таблица-скелетон 6 строк; при фоновом обновлении — тонкий прогресс, контент не мигает.
- **Empty:** «Пока нет заказов» (для фильтра — «Нет заказов со статусом „Готовы“»), иллюстрация, без кнопок.
- **Error сети:** тост «Не удалось обновить список» + данные последней успешной загрузки с меткой «Обновлено в 14:32».
- **Offline:** баннер «Нет соединения», авто-обновление на паузе, кнопки смены статуса disabled.
- Конфликт (заказ уже изменён другим админом): тост «Заказ обновлён другим устройством», строка перерисовывается.

**Haptics:** админ обычно на desktop — haptics не используется; на телефоне смена статуса → `notificationOccurred('success')`.

## Контрасты (WCAG AA, Lighthouse a11y ≥95)

- Основной текст `ink` на `bg` / `surface`: ≥12:1 в обеих темах.
- Вторичный `muted`: ≥4.5:1 (проверен на самом светлом фоне светлой темы).
- Кнопка primary: светлая — белый на `#C2410C` ≈ 5.0:1; тёмная — тёмный текст на `#E8632C` ≈ 5.8:1.
- Статусы/бейджи: цвет + текстовая подпись (никогда только цвет).
- Все интерактивы — фокус-кольцо `focus-visible`, изображения — `alt`, степперы и IconButton — `aria-label`, суммы — `aria-live="polite"` на cart-bar.
